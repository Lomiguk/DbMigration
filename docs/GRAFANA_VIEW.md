График разделился на три линии, потому что виртуальная машина Java (JVM) делит Heap (кучу) на несколько пулов памяти для оптимизации сборки мусора (Garbage Collection). Обычно это:

G1 Eden Space (где создаются новые объекты, они живут очень недолго).

G1 Survivor Space (объекты, которые пережили первую сборку мусора).

G1 Old Gen (старые объекты, которые живут долго — например, ваш кэш mapping_cache).

Prometheus отдает метрику jvm_memory_used_bytes{area="heap"} сразу для всех трех пулов, а Grafana рисует линию для каждого из них
